 Strong Vs Loose Typinge   
C# JavaScript

In C# compiler Inferred the type of the variable and can not be assigned to different type after that. Compiler produces an error if you try to so.
var customer = new Customer();
In JavaScript we don't have strong typing so we can assign it to any type time later. In JavaScript we don't have compile time error and if anything goes wrong it will break on run time only.
Strong Typing and OOP: Inheritance and implementation are part of type identity. Loose Typing and OOP: Type is less important, but the shape is important. This is what they call Classic Duck Typing.
Duck Typing: If it walks like a duck, quack like a duck and smells like a duck... It's a Duck. We don't need identity of duck to proof and characteristics are sufficient.

 Global Scope   
In C# all the code take place inside the class. We have properties and method in a class and execution starts with Main function.

JavaScript interpreted so all your code take place in the root of the file or we can say in global scope. So we can define a variable... define a function... and make call to that function... all in one place.
// JavaScript var x = 1; function myFunction(arg1, arg2) {   //... } myFunction(1,x);
 Type Coalescing   
Type Coalescing: mix together different elements to form one;
In C# when joining two different types object we need to define typecasting (implicitly or explicitly), but JavaScript wants to coalesce values by default. For example in C# if we want to add one string and one integer we need to convert that integer first into string.
// C# "akash " + "sharma"; // result = "akash sharma" "akash " + 219.ToString; // have to call ToString method to convert in string first. //OR you can call String.Concat("akash ", 219) to force coalescing automatically.
Where as in C# you either have to convert the value or have to force in some way to automatically coalesce value for you, but in JavaScript it is a default behavior. If JavaScript finds operation between two different type, it convert one value (logically).
// JavaScript "akash " + "sharma"; // result = "akash sharma" "akash " + 219; // result = "akash 219"... without casting // same with other types as well "akash " + true; // result = "akash true"
Problem area for C# programmers is when comparing two types... JavaScript do comparison with coalescing... So we get Identically Equality Operators (===, !==) similar to .Equal() method in C# to determine the equality without coalescing.
1 == "1"; // result = true 1 === "1"; // result = false
