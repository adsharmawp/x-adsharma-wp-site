JavaScript functions look like C# function, but they are not exact same In JavaScript function are like C# methods and C# classes both and depend on the context.
 JavaScript Function Vs C# method   
Let's have an example:
// defining function     function foo(args1, args2) { } // calling function foo(1,2);
It looks same as C#; but in JavaScript you can call same method with no or less argument or with greater number of arguments:
foo(); foo(1); foo(1,2,3);
So, what's going here? Function in JavaScript are not differentiate by argument type or list. This also mean we can nor overload a method. If you define 2 same name method with diffrent argument then 2nd definition interpreted will take place in memory and 1st will be vanish. So why arguments? - for easy access of your variables;
function foo(a, b, c){     document.write(a);     document.write(b);     document.write(c); } foo(1,2,3); // Output: 1,2,3 foo(1,2); // Output: 1,2,undefined foo(1,2,3,4); // Output: 1,2,3 (But wait where is 4?)
How to stop printing 'undefined'?
function foo(a, b, c){     if(a) document.write(a);     if(b) document.write(b);     if(c) document.write(c); }
How to know about arguments? argument object: there is a argument object available inside functions only:
function foo(a,b,c){     document.write(arguments.length); } foo(1,2,3); // Output: "3" foo(1,2); // Output: "2" foo(1,2,3,4); // Output: "4"
Accessing all arguments in JavaScript:
function foo(){     for (var i = 0; i < arguments.length; i++)     document.write(arguments[i]); } foo(6,7,8); // Output: 6,7,8
foreach style; but gives index insted of actual object;
function foo(){     for (var arg in arguments)     document.write(arg); // Output: 0,1,3     document.write(arguments[arg]); // Output: 6,7,8 } foo(6,7,8);
 JavaScript Function like Object   
JavaScript function can behave like any other object.
function foo(a,b,c){ alert('hi'); } document.write(foo.length); // Output: 3 [based on parameters] document.write(foo.toString()); // Output: whole definition of function.
So you define a function and assign it to an object same like ‘anonymous delegate’ in C#
var foo = function (a,b,c){ alert('hi'); } var l = foo.length; // 3 : same like above properties foo(1,2,3); // Calling
 JavaScript Function like Class   
JavaScript function can be used as Class:
function Student(fname, lname) {     this.firstName = fname;     this.lastName = lname;          this.FullName = function () {         document.write(this.firstName + " " + this.lastName);     } } var s = new Student("akash", "sharma"); s.FullName();
If we use var instead of this the variable will be treated as private member of the class. 'this' represent the instance of the class or in other words 'owner'. However scope is different (engine need to know the scope so that variable can be collected by GC). So in JavaScript scope of a variable is based on function and not on { }.
var obj = {     name: "rohit",     myFunc1: function () {     if (true) {         var age = "28";     }     document.write(this.name);     document.write(age);     } } obj.myFunc1(); // Output: rohit28
