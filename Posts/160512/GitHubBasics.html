 What is Git?   
The core of GitHub is Git @ https://git-scm.com/ (a distributed version control system).

 Cetralized Vs. Distributed VCS   
Cetralized VCS: Maintain central repository and individual can 'check-out' files and 'check-in' once done with changes.
Distributed VCS: Each client keeps a local mirror repository; Individual change 'pull' or 'push' changes from other repos or sync changes within multiple repos.

When working with centralized version control system; you might have encountered a situation when you have to create multiple copy of a project (might be due to many different issues) on your local machine and once you done with the changes you merge them back to master copy manually and commit the changes. However VCS (Version Control System) can make this process very easy and unfortunately that is available for main branch to VCS only. But with Git this is available on every terminal because every terminal is now a VCS itself. This can help great while you are tracking changes (locally) and prevent overwriting or inadvertently deleting files.

 How Git works...   
Git works by taking snap-shots each time you save (commit) current state. Every new commit points back to older commit… and each commit represent by a 40 character hash like ‘a7fa2354...’; but Git also provides a short 7 character hash like ‘a7fa235’ for easier to identify and remember.
If you initialize a folder as ‘Git Repo’ git creates some hidden files just like SVN to track all the changes.

* There 3 basic areas of the project:
  • Working directory: Current state of your files.
  • Staging area: Contains index of every changes for next commit.
  • Git Repo: Once you commit changes are in Git Repo.
* Any time each file will be in any of the 3 state:
  • Unmodified: Not changed science last commit.
  • Modified: File changed sicence last commit.
  • Stage: Staging a file mean ready for next commit.

 Delete a branch   
To delete a exiting branch:
C:\github\git-first [master]> git branch -d branch1
C:\github\git-first [master]> git branch -d branch2
C:\github\git-first [master]> git branch -d branch3
C:\github\git-first [master]> git branch -d branch4

But if there are changes in the branch that are not merged to main branch till yet; git will not allow you to delte the branch. So in that situation you either need to merge the changes or you can simply hard delete a branch by using -D insted of -d.
